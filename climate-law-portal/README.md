# Climate Law Intelligence Portal — Pakistan

A free, open Django-based search engine for Pakistan's climate &
environmental case law. Stores **5,000+** environmental tribunal cases,
exposes a fast keyword search across title / summary / full text, and
deploys to Render's free tier in one click.

---

## Features

- 🔎  Keyword search across title, court, summary, full text and tags.
- 🎚  Year and violation-type filters; sort by relevance / newest / oldest / title.
- 📄  Stable detail page for each case (`/case/<id>/<slug>/`).
- 🛠  Full Django admin (CRUD, filters, search) at `/admin/`.
- 📦  Bulk ingestion: `.txt` corpus, CSV, or JSON.
- 🚀  Render-ready: `Procfile`, `render.yaml`, `build.sh`, `gunicorn.conf.py`.

## Quick start (local)

```bash
python -m venv .venv
.venv\Scripts\activate                # Windows
# source .venv/bin/activate           # macOS / Linux

pip install -r requirements.txt

python manage.py migrate
python manage.py createsuperuser      # optional — admin user
python manage.py ingest_cases         # ingests all .txt files in ./all_text_cases
python manage.py runserver
```

Open `http://127.0.0.1:8000/`.

### Importing data

```bash
# 5,000+ .txt files in ./all_text_cases
python manage.py ingest_cases

# CSV
python manage.py import_cases_csv path/to/cases.csv

# JSON
python manage.py import_cases_json path/to/cases.json
```

CSV / JSON columns:
`case_title`, `court`, `country`, `year`, `violation_type`,
`summary`, `full_text`, `tags`, `source_file`.

## Deploying to Render (free)

1. Push this repo to GitHub.
2. On https://dashboard.render.com → **New** → **Blueprint** → pick the repo.
   `render.yaml` is auto-detected.
3. Render builds with `./build.sh` (installs deps, runs `collectstatic` and
   `migrate`) and boots gunicorn via `Procfile` / `render.yaml`.
4. Visit the generated `*.onrender.com` URL.
5. (Optional) populate the database from the Render shell:
   ```bash
   python manage.py createsuperuser
   python manage.py ingest_cases --path /opt/render/project/src/all_text_cases
   ```

### Required environment variables

| Variable                | Example                                  |
| ----------------------- | ---------------------------------------- |
| `DJANGO_SECRET_KEY`     | auto-generated by `render.yaml`          |
| `DJANGO_DEBUG`          | `0` in production                        |
| `DJANGO_ALLOWED_HOSTS`  | `.onrender.com`                          |
| `DATABASE_URL`          | optional — falls back to local SQLite    |
| `CASES_PER_PAGE`        | `20` (default)                           |

> ⚠️ Render's free tier has an ephemeral filesystem. SQLite data does not
> survive deploys. For production, attach a free PostgreSQL instance and
> set `DATABASE_URL` — the project picks it up automatically via
> `dj-database-url`.
